

1. **git init**: Initializes a new Git repository in the current directory.
   
2. **git clone [url]**: Clones a repository from a remote server to your local machine.

3. **git add [file(s)]**: Adds file changes to the staging area in preparation for committing.

4. **git commit -m "[message]"**: Commits the staged changes to the local repository with a descriptive message.

5. **git status**: Displays the current state of the working directory and staging area.

6. **git diff**: Shows the differences between the working directory, staging area, and the last commit.

7. **git push**: Uploads local repository commits to a remote repository.

8. **git pull**: Fetches changes from a remote repository and merges them into the local branch.

9. **git fetch**: Downloads objects and refs from another repository without merging them into your local branches.

10. **git branch**: Lists existing branches, creates new branches, or deletes branches.

11. **git checkout [branch|file]**: Switches branches or restores files from the staging area or commit.

12. **git merge [branch]**: Merges changes from one branch into another.

13. **git log**: Displays a chronological list of commits.

14. **git reset [file]**: Resets the staging area to the last commit while leaving the working directory unchanged.

15. **git stash**: Temporarily shelves changes that aren't ready to be committed, allowing you to work on other tasks.

16. **git tag**: Lists, creates, or deletes tags to mark specific points in history as important.

17. **git remote**: Manages connections to remote repositories.

18. **git rebase**: Moves or combines a sequence of commits to a new base commit.
